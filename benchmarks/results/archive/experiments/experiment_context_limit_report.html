Agent Context Limit Experiment

Testing whether passing raw structured context (20K tokens) vs LLM-summarized context (2-3K chars) improves GRAIL agent answer quality. 5 patient-language questions, 5 systems compared.

Root Cause Found

The mini-agent summarizer is the bottleneck, not the search

19-25K

Cascade raw context (tokens)

Every cascade search produces 19,000-25,000 tokens of structured context (entities + relationships + communities + text units). This exceeds even the 20K limit.

2-3K

After mini-agent summary (chars)

The summarizer compresses 72-91K chars down to 1.8-3.5K chars — a 96% reduction. Article numbers, specific amounts, and procedural details are lost.

4-5/5

Direct Cascade scores

When cascade search sends its full context directly to the LLM (no agent wrapper), it scores 4-5/5 on gold keywords. The retrieval works perfectly.

0-2/5

GRAIL Agent 8K scores

After summarization, the agent only scores 0-4/5. The information exists in the search results but gets discarded before the agent sees it.

Conclusions

What the experiment proves and what to do next

Cascade retrieval is excellent

Direct Cascade scores 4.0/5 average — matching RAG. The entity graph + text rescue combination works. The structured context (entities, relationships, communities, text units) IS more valuable than raw chunks alone.

The mini-agent destroys the advantage

GRAIL Agent 8K scores 1.8/5 vs Direct Cascade's 4.0/5. The same retrieval, same context, but a lossy LLM summarizer in between drops 2.2 points. This is the single biggest bottleneck.

20K limit helps but isn't enough

GRAIL 20K scores 2.8/5 — a +55% improvement over 8K but still below Direct Cascade (4.0). The cascade contexts are 19-25K tokens, so even 20K triggers summarization on 4/5 questions.

Recommended fix

Set agent_tool_context_limit to 30K tokens (or remove it entirely). Let the agent LLM handle the full structured context. Modern models (128K context) can easily process 25K input tokens. The agent is already an LLM — it doesn't need another LLM to pre-digest its input.
