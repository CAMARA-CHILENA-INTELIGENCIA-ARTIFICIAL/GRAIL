Key Findings

What this benchmark reveals

GRAIL is faster and more reliable

Average 11.3s vs RAG's 31.1s. GRAIL uses 2.3 LLM calls per question vs RAG's 3.7. The structured context (entities + relationships + text units) gives the LLM everything in one pass. RAG often needs multiple search refinements.

Tool diversity works

The agent uses local_search for entity-focused queries (Q02, Q09, Q11-Q14), cascade for text-detail queries (Q01, Q03, Q06-Q08, Q10), and even document_search (Q05). This strategic tool selection is GRAIL's unique advantage over single-tool RAG.

RAG fails on complex contexts

RAG returned empty on Q01, Q07, Q10 — all cases where the thinking model exhausted its token budget processing large raw chunk contexts (13K+ tokens across multiple searches). GRAIL's single-pass structured context avoids this.

Q15 remains hard for GRAIL

GRAIL failed on Q15 (when does protection end?) — made 3 cascade searches consuming 33.8K tokens but couldn't synthesize an answer. This is a question where the answer (D54 Art.10) requires very specific article-level detail that cascade finds but the synthesis fails under thinking-model token pressure.
