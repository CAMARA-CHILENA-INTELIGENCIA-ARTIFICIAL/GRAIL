# GRAIL — Concept Summary for Logo Image Generation

GRAIL (Graph RAG with Advanced Integration and Learning) is an open-source Python library that turns large, messy document collections into queryable knowledge graphs. It is a hardened fork of Microsoft GraphRAG built by Benjamin Gonzalez Guerrero under the Cámara Chilena de Inteligencia Artificial / Nirvai. The pipeline ingests files, uses LLMs to extract entities and the relationships between them, clusters those entities into hierarchical "communities" via Leiden graph clustering, and writes narrative reports for each community — so a corpus that started as raw text becomes a structured, navigable map of who, what, and how things relate.

What makes GRAIL distinctive is its multi-mode retrieval. Instead of a single RAG path, it offers five search behaviors over the same graph: local (entity-anchored), cascade (graph + BM25 text rescue for facts the graph missed), global (map-reduce over community reports for thematic questions), document (scoped to one source file), and agent (an LLM tool-caller that picks the right mode per question). Every answer carries citations back to the original documents. The library is built for honest cost tracking, incremental graph updates as new files arrive, optional typed relationships, and self-hosted as well as cloud LLM endpoints — so it reads as serious infrastructure, not a chatbot wrapper.

The name "GRAIL" is a deliberate reference to the quest for knowledge: a graph as the artefact, and retrieval as the search. Visually, the project lives at the intersection of scholarly research, open-source engineering, and graph theory — its natural symbolic vocabulary includes nodes and edges, hierarchical clustering, layered communities, document-to-graph transformations, and the idea of a held, found, or revealed object that contains organized knowledge. The mark should feel precise, considered, and a little reverent — closer to an academic library or a research instrument than to a generic AI startup glyph.
